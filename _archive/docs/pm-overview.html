// 02 — Architecture

Clean 4-Layer Architecture

HTTP Request

Browser → Nginx → public/index.php

↓

Controller

HTTP adapter only — validates input, returns response. Zero query logic.

↓

Service Layer

Business rules live here — ownership checks, approval transitions, role guards.

↓

Repository

All database queries. Implements ProductRepositoryInterface contract.

↓

Eloquent Model → MySQL

Product, User — parameterised queries, soft deletes, eager loading


architecture.php
// Request hits ProductController@store (POST /products)

ProductRequest       ──→  authorize() checks role, prepareForValidation() strips XSS
ProductController    ──→  calls service, returns redirect. No query logic here.
ProductService       ──→  stamps created_by, sets status='pending', checks canModify()
ProductRepository    ──→  Product::create($data) — the only place queries are written
MySQL                ──→  row inserted, id returned

// Email notification (admin alerted to new pending product)
ProductService  ──→  Mail::to(admin)->send(new ProductSubmittedMail($product))

// 03 — Features

What Each Role Can Do

🔐 Admin — Full Control
  • ✓ Approve or reject any pending product
  • ✓ Edit or soft-delete any product regardless of creator
  • ✓ Create and manage user accounts
  • ✓ Assign roles (admin / product_manager / customer)
  • ✓ View all orders across all customers
  • ✓ Update order status (triggers customer email)
  • ✓ Email sent to manager on approval/rejection
📋 Product Manager
  • ✓ Create products with Quill rich-text description
  • ✓ Set price, category, stock quantity
  • ✓ Products enter pending state automatically
  • ✓ Edit and soft-delete own products only
  • ✓ Full-text search across title + description
  • ✓ Pagination on product list (12 per page)
  • ✓ Receive email notification on approve/reject
🛡️ Security Built-In
  • ✓ XSS sanitisation on all rich-text input (strip_tags allow-list)
  • ✓ CSRF token on every state-changing form
  • ✓ Rate-limited login — 5 attempts then lockout
  • ✓ Role checked at route level AND in service layer
  • ✓ Ownership check — manager can't edit others' products
  • ✓ Soft deletes — products recoverable after deletion
⚙️ Developer Experience
  • ✓ bash deploy.sh — full setup in one command
  • ✓ Makefile: make lint, make test, make fresh, make shell
  • ✓ MailHog UI at localhost:8025 — see all emails instantly
  • ✓ Laravel Pint enforces PSR-12 style automatically
  • ✓ PHPUnit test suite wired up (make test)

// 05 — Security

Security Measures

ThreatHow it is stopped
SQL InjectionEloquent parameterised query builder throughout. No raw SQL built from user input.
Stored XSSProductRequest::prepareForValidation() calls strip_tags() with a safe-tag allow-list before any data is validated or saved.
CSRF@csrf on every POST/PATCH/DELETE form. VerifyCsrfToken middleware rejects missing tokens.
Brute Force LoginRateLimiter — 5 attempts per email+IP, then throttle. Lockout event fired.
Privilege EscalationCheckRole middleware blocks at route level. ProductService::canModify() enforces ownership at object level. abort(403) on failure.
Mass Assignment$fillable whitelist on every model. Attackers cannot inject role=admin via a crafted form submission.
Session Fixation$request->session()->regenerate() on login. invalidate() + regenerateToken() on logout.
Sensitive FilesNginx denies /dotfile requests. .env is gitignored. Docker image never exposes .env.
