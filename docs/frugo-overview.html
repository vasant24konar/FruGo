// 02 — Architecture

How a Shop Request Travels Through the App

shop-request-lifecycle.php
// Customer visits GET /shop?search=mango&page=2

Browser              ──→  Nginx (:8080)          // serves CSS/JS directly, forwards PHP
Nginx                ──→  public/index.php       // Laravel front controller
// Middleware runs: EncryptCookies → StartSession → VerifyCsrfToken
// auth middleware checks if logged in (OTP session active)

Route::get('/shop', [ShopController::class, 'index'])

ShopController       ──→  reads $search from URL, calls repository
ProductRepository    ──→  Product::approved()->search($q)->paginate(12)
MySQL                ──→  returns matching approved products, page 2

// Response travels back
$products  ──→  ShopController  ──→  view('shop.index')  ──→  HTML to browser

// Cart count shown in navbar reads from session — zero DB queries
array_sum($request->session()->get('cart', []))

How the Two Core Flows Work

🔑 OTP Login — End to End
1
Customer enters emailOtpLoginController validates format
2
OTP generated & saved6 digits, stored in otp_tokens with expires_at
3
Email dispatchedOtpMail sent via MailHog (dev) or real SMTP (prod)
4
Customer enters code6 digit boxes, auto-focus, paste support
5
Token deleted immediatelyCannot be reused. User::firstOrCreate() auto-registers new customers.

Add to cartCartController stores product ID + qty in session array

CheckoutAddress form, order total calculated from session cart

Order createdOrder + OrderItems saved to DB, cart session cleared
5
Track on orders pageFilter by status tab, paginated with ?status= preserved

// 05 — Security

Security Measures

ThreatHow it is stopped
OTP ReplayToken deleted immediately on successful verification. Hard 10-minute expiry enforced by expires_at column check.
SQL InjectionAll queries through Eloquent parameterised builder — user-supplied ?search= value is never concatenated into raw SQL.
CSRF@csrf on every cart/checkout/order form. VerifyCsrfToken middleware rejects requests without a valid token.
Stored XSSProduct descriptions strip_tagged on save. Blade {{ }} HTML-encodes all output. {!! !!} only used on pre-sanitised fields.
Session Fixationsession()->regenerate() on OTP login success. invalidate() + regenerateToken() on logout.
Privilege EscalationCustomers cannot access /products, /admin routes. CheckRole middleware + auth guard enforced on all protected routes.
Sensitive Env Vars.env gitignored. .env.example committed (no secrets). Docker image never exposes .env. Vercel env vars set via dashboard.

// 06 — Deployment

Local & Vercel Deployment

BASH — Local Docker setup
# Clone and enter
git clone https://github.com/vasant24konar/FruGo.git && cd FruGo

# Copy env (no edits needed for local Docker)
cp .env.example .env && bash deploy.sh

http://localhost:8080      # FruGo shop
http://localhost:8025      # MailHog — see OTP emails here

Vercel Production: Requires external MySQL (PlanetScale) + transactional SMTP (Resend/Mailgun). Set these env vars in Vercel dashboard:

Required

APP_KEY, APP_ENV=production

Generate key with php artisan key:generate --show

Database

DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD

Point to PlanetScale or Railway MySQL

Session/Cache

SESSION_DRIVER=cookie, CACHE_STORE=array

Serverless needs no file writes

Mail

MAIL_MAILER=smtp + host/port/user/pass

Use Resend, Mailgun, or SendGrid
